# Architecture

## High-Level Design

### Ingestion Pipeline

```
.txt files on disk
      |
      v
load_text_files(directory, chunk_size=500)
      |
      v
[chunk_1, chunk_2, ..., chunk_N]   (list[str])
      |
      v
RAGEngine.add_documents(texts, ids)
      |
      v
ollama.embed(model="nomic-embed-text", input=chunk)
      |
      v
ChromaDB.collection.add(embeddings, documents, ids)
      |
      v
Persisted to ./chroma_db/
```

### Query Pipeline

```
User question (HTTP POST /query)
      |
      v
RAGEngine.generate_answer(question)
      |
      v
ollama.embed(model="nomic-embed-text", input=question)
      |
      v
ChromaDB.collection.query(query_embeddings, n_results=3)
      |
      v
Top-N matching document chunks (context)
      |
      v
Prompt assembly: "Answer based on context:\n{context}\nQuestion: {question}"
      |
      v
ollama.generate(model="qwen2.5-coder:3b", prompt=prompt)
      |
      v
JSON response: { question, answer, model }
```

## Module Responsibilities

| Module | Role |
|---|---|
| `document_loader.py` | Read `.txt` files from disk, split into fixed-size character chunks |
| `rag_engine.py` | Owns ChromaDB client and collection; wraps embed and generate calls |
| `api_server.py` | Flask app factory; wires HTTP routes to RAGEngine methods |
| `__main__.py` | Process entrypoint; loads `.env`, instantiates app, starts dev server |

## Flask Factory Pattern

`create_app()` follows the Flask application factory pattern. It accepts an optional `RAGEngine` instance:

```python
def create_app(rag: RAGEngine | None = None) -> Flask:
```

When `rag` is `None`, it constructs one from environment variables. When a `RAGEngine` is passed explicitly — as in tests — the factory uses it directly. This decouples the HTTP layer from infrastructure and makes the server testable without Ollama or ChromaDB on disk.

## Upgrade Path

The starter stack is a single Flask process on port 5000. Running `scripts/upgrade.sh` extends it to a dual-process production architecture:

```
                    Flask (port 5000)
                    - GET /chat      (browser chat UI)
                    - POST /query    (JSON API)
                    - GET /health

                    FastAPI (port 7860)
                    - POST /api/stream   (SSE streaming)
                    - POST /api/ingest
                    - GET  /api/health
```

Both processes share the same `RAGEngine`, ChromaDB path, and Ollama backend. They can run behind a reverse proxy (nginx or Caddy) if external access is needed.

The upgrade script adds these files idempotently (skips if already present):

- `src/rag_chatbot/fastapi_app.py` — FastAPI backend with SSE streaming
- `src/rag_chatbot/pdf_loader.py` — PDF ingestion via pymupdf
- `src/rag_chatbot/chat_history.py` — SQLite-backed chat session storage
- `src/rag_chatbot/templates/chat.html` — Browser chat UI
- `services/ai-stack-api.service` — systemd unit for FastAPI
- `services/ai-stack-web.service` — systemd unit for Flask

## Data Flow Summary

All data stays local. No bytes leave the machine:

- Documents are read from disk by `load_text_files()` or `load_pdf()`
- Embeddings are generated by Ollama running `nomic-embed-text` locally
- Vectors and document text are persisted in ChromaDB on disk (`./chroma_db/`)
- Answers are generated by Ollama running `qwen2.5-coder:3b` locally
- Chat history (after upgrade) is stored in SQLite (`./chat.db`)
